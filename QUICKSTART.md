# Quick Start Guide

Get the Topic Clusterer up and running in 5 minutes.

## 1. Setup (1 minute)

### Option A: Using Virtual Environment (Recommended)

```bash
# Navigate to project directory
cd C:\Users\ridhs\Downloads\Projects\Topic\ Clusterer

# Create virtual environment
python -m venv venv

# Activate it
venv\Scripts\activate  # Windows
# or: source venv/bin/activate  # macOS/Linux

# Install dependencies
pip install -r requirements.txt
```

### Option B: Direct Installation
```bash
pip install -r requirements.txt
```

## 2. Configure API Key (1 minute)

1. Get your Anthropic API key from https://console.anthropic.com/
2. Create `.env` file in the project root:
   ```
   ANTHROPIC_API_KEY=sk-ant-your-key-here
   ```
3. Or copy and edit `.env.example`:
   ```bash
   copy .env.example .env  # Windows
   # or: cp .env.example .env  # macOS/Linux
   ```

## 3. Run the Pipeline (3 minutes)

```bash
python topic_clusterer.py
```

This will:
- Scrape 100 recent headlines
- Generate embeddings
- Cluster articles
- Label clusters
- Create visualizations

## 4. View Results

Three output files are created:

### Static Visualization
Open `topic_clusters.png` - shows all clusters as a scatter plot

### Interactive Map (Recommended!)
Open `topic_clusters.html` in your browser:
- Hover over points to see full headlines
- Click legend to toggle clusters
- Zoom and pan to explore
- Color-coded by topic

### Data Export
Open `clustering_results.json` to see:
- All articles grouped by cluster
- Cluster labels and sizes
- Full article metadata

## Quick Configuration

Edit DBSCAN parameters in `topic_clusterer.py` line ~395:

```python
# More clusters (stricter)
labels, n_clusters = cluster_with_dbscan(embeddings, eps=0.4, min_samples=2)

# Fewer clusters (broader)
labels, n_clusters = cluster_with_dbscan(embeddings, eps=0.8, min_samples=1)

# Default (balanced)
labels, n_clusters = cluster_with_dbscan(embeddings, eps=0.6, min_samples=2)
```

## Understanding the Output

### What are clusters?
Groups of semantically similar news articles automatically identified by the algorithm.

### What do the colors mean?
Each color represents a different topic cluster. Articles with the same color are about similar topics.

### What are the noise points (black X)?
Articles too different from others to fit cleanly into any cluster - usually miscellaneous items.

### What do the labels mean?
Generated by Claude based on the articles in each cluster. Examples: "Technology News", "Climate Updates", etc.

## Troubleshooting

### "ANTHROPIC_API_KEY not found"
- Check `.env` file exists in project root
- Verify key format: `ANTHROPIC_API_KEY=sk-ant-...`
- Restart Python after creating `.env`

### "Connection timeout"
- Check internet connection
- Some RSS feeds may be temporarily down
- Script continues with available feeds

### "No clusters found"
- Increase `eps` parameter (0.7 or 0.8)
- Articles may be too diverse

### "Too many noise points"
- Decrease `eps` parameter (0.4 or 0.5)
- Increase `min_samples`

### Port already in use (if using interactive plots)
- Close other Python processes
- Change port in code if needed

## Tips & Tricks

1. **Faster Processing**: Reduce `num_articles` to 20-30 for testing
   ```python
   articles = scrape_rss_feeds(num_articles=30)
   ```

2. **Better Results**: Increase to 200+ articles for more coherent clusters

3. **Dark Mode**: Use Plotly's dark template
   ```python
   fig.update_layout(template='plotly_dark')
   ```

4. **Export as Image**: Right-click on HTML plot → "Save image as"

5. **Batch Multiple Runs**: Create a loop to cluster different RSS sources
